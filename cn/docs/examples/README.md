# 协作会话示例

此目录包含现实的、端到端的会话记录，展示游戏工作室智能体架构在实践中如何工作。每个示例演示**协作工作流程**，其中智能体提出问题、呈现选项，并在自主生成内容之前等待用户批准。

---

## 📚 **可用示例**

### [会话：设计制作系统](session-design-crafting-system.md)
**类型：** 设计
**智能体：** game-designer
**时长：** 约 45 分钟（12 轮）
**复杂度：** 中等

**场景：**
独立开发者需要设计一个服务于支柱 2（"通过实验的紧急发现"）的制作系统。智能体通过问答引导他们，呈现 3 个带游戏理论分析的设计选项，合并用户修改，并在每个步骤获得批准后迭代起草 GDD。

**关键协作时刻：**
- 智能体 upfront 提出 5 个澄清问题
- 呈现 3 个带优缺点和 MDA 对齐的不同选项
- 用户修改推荐选项，智能体立即合并
- 主动标记边缘情况（"如果非配方组合呢？"）
- 每个 GDD 部分在进入下一个之前显示供批准
- 在创建文件之前明确"我可以写到 [文件] 吗？"

**学习：**
- 设计智能体如何询问目标和约束
- 如何使用游戏设计理论（MDA、SDT、Bartle）呈现选项
- 如何逐节迭代草案
- 何时委托给专家（systems-designer, economy-designer）

### [会话：实现战斗伤害计算](session-implement-combat-damage.md)
**类型：** 实现
**智能体：** gameplay-programmer
**时长：** 约 30 分钟（10 轮）
**复杂度：** 低-中等

**场景：**
用户有完整的设计文档，想要实现伤害计算。智能体读取规范，识别 7 个模糊/差距点，提出澄清问题，为批准提出架构，实现并执行规则，主动编写测试。

**关键协作时刻：**
- 智能体首先阅读设计文档，识别 7 个规范模糊点
- 在实现之前用代码样本提出架构
- 用户要求类型安全，智能体改进并重新提出
- 规则捕获问题（硬编码值），智能体透明修复
- 遵循验证驱动开发主动编写测试
- 智能体提供下一步选项而不是假设

**学习：**
- 实现智能体如何在编码前澄清规范
- 如何用代码样本为批准提出架构
- 规则如何自动执行标准
- 如何处理规范差距（问，不要假设）
- 验证驱动开发（测试证明它有效）

### [会话：范围危机 - 战略决策制定](session-scope-crisis-decision.md)
**类型：** 战略决策
**智能体：** creative-director
**时长：** 约 25 分钟（8 轮）
**复杂度：** 高

**场景：**
独立开发者面临危机：2 周后 Alpha 里程碑，制作系统需要 3 周，投资者演示至关重要。创意总监收集上下文，正确框架决策，用诚实权衡分析呈现 3 个战略选项，提出推荐但 defer 给用户，然后用 ADR 和演示脚本记录决策。

**关键协作时刻：**
- 智能体在提出解决方案之前阅读上下文文档
- 提出 5 个问题以了解决策约束
- 正确框架决策（赌注是什么，评估标准）
- 用风险分析呈现 3 个选项和历史先例
- 提出强烈推荐但明确地："这是你的决定"
- 记录决策 + 提供演示脚本以支持用户

**学习：**
- 领导智能体如何框架战略决策
- 如何用权衡分析呈现选项
- 如何在推荐中使用游戏开发先例和理论
