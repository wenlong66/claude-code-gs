# 示例会话：使用 /design-system 编写 GDD

**上下文：** 单人独立开发者，项目第 3 周。`/map-systems` 已经运行，并生成了 `design/gdd/systems-index.md`。移动系统被列为 MVP 层级、基础优先级。

**起始状态：**
- `design/gdd/game-concept.md` 已存在（生存制作游戏，2D，Godot 4.6）
- `design/gdd/systems-index.md` 已存在（识别出 15 个系统，移动是最先设计的）
- 还没有编写任何单系统 GDD
- 移动系统没有上游依赖（它是基础）

**会话时长：** 约 60 分钟
**轮数：** 14
**结果：** 完整的 `design/gdd/movement-system.md`，包含全部 8 个必需部分，并已写入文件，可交给 `/design-review`

## 第 5 轮：第 3 节——详细规则

**Agent：**
> **第 3 节草稿——详细规则：**
>
> > **基础状态：**
> > - **Walk**（默认）：全方向控制，中等速度，低耐力消耗
> > - **Run**（按住 Shift）：更快，每秒中等耐力消耗——耐力降到 0 时自动取消
> > - **Roll/Dodge**（点按 Space）：沿移动方向快速突进，短暂无敌帧，高耐力消耗（固定值）
> > - **Idle**：没有移动输入，耐力以最大速率恢复
> >
> > **地形修正：**
> > - 普通地面：1.0x 速度乘数，标准摩擦
> > - 沙地/松土：0.8x 速度，1.1x 耐力消耗（更难移动）
> > - 滑岩（湿滑）：1.1x 速度，减小停止摩擦（玩家会滑过去）
> > - 浅水（到脚踝）：0.7x 速度，不改摩擦
> > - 深水：禁用移动，改为游泳（单独系统，范围外）
> >
> > **碰撞：**
> > - 使用 CharacterBody2D + move_and_slide()；启用墙面滑动
> > - 45° 以下斜坡可正常行走；大于 45° 的斜坡会阻挡
> > - 支持单向平台（按 Down + Jump 下穿）

## 第 6 轮：第 4 节——公式

**Agent：**
> **第 4 节草稿——公式：**
>
> 我先定义变量，再给公式：
>
> | 变量 | 类型 | 说明 |
> |----------|------|-------------|
> | `base_walk_speed` | float | 基础步行速度，单位 px/s（可调） |
> | `run_multiplier` | float | 奔跑速度 = 步行 × 这个值 |
> | `terrain_speed_mod` | float | 每种地形的乘数（0.7–1.1） |
> | `stamina_drain_walk` | float | 步行时每秒消耗的耐力 |
> | `stamina_drain_run` | float | 奔跑时每秒消耗的耐力 |
> | `stamina_cost_roll` | float | 每次翻滚的固定耐力消耗 |
> | `terrain_drain_mod` | float | 每种地形的耐力消耗乘数 |
>
> **有效速度：**
> ```
> effective_speed = base_walk_speed × run_multiplier（如果在奔跑，否则为 1.0）× terrain_speed_mod
> ```
>
> **耐力消耗（每秒）：**
> ```
> drain_per_sec = stamina_drain_walk（如果在奔跑则用 stamina_drain_run）× terrain_drain_mod
> ```
>
> **翻滚耐力消耗（固定值）：**
> ```
> stamina_after_roll = current_stamina - stamina_cost_roll
> （钳制到 0 —— 翻滚仍然执行，但玩家落地时会是 0 耐力）
> ```
>
> 变量名和公式都没问题吗？

**User：**
> 看起来对。写进去。

## 第 7-9 轮：第 5、6、7 节

**Agent：**
>（这几轮保持相同模式——智能体起草，用户批准或要求小调整，智能体在进入下一节前先把每一节写进文件。这里简写。）

**第 5 节——边缘情况** 包含：
- 玩家撞墙翻滚（动画照常完成，但没有位移加成）
- 耐力在奔跑中降到 0（切回步行，不会突然卡顿）
- 两种地形重叠（取较低乘数）
- 在滑岩上翻滚（仍然套用滑岩停止摩擦，但不影响翻滚本身）

**第 6 节——依赖关系** 包含：
- **Stamina 系统**（下游）：移动暴露 `on_stamina_event(type: String, amount: float)` 信号
- **Inventory 系统**（下游）：监听 `carrying_heavy_object_changed(is_heavy: bool)` 信号
- **输入系统**：移动方向来自 `InputMap`——标准 Godot action 名称（`ui_left`、`ui_right` 等）
- **地形层**：tile 元数据驱动地形修正查询

**第 7 节——调优旋钮** 包含：
- `base_walk_speed`（默认：120 px/s）
- `run_multiplier`（默认：1.7）
- 各地形的速度和耐力消耗修正（4 种地形表）
- `stamina_drain_walk`、`stamina_drain_run`、`stamina_cost_roll`

## 这个示例展示了什么

- **增量写文件**：每一节在批准后立刻写盘——会话崩溃最多丢失一个正在写的部分
- **模板填充**：GDD 模板的 8 个部分通过对话变成具体内容，而不是一次性大生成
- **依赖信号**：智能体在 Dependencies 部分主动暴露下游信号契约（stamina、inventory）
- **技术可行性预检**：智能体在设计开始前就注意到 Jolt 物理变化，而不是等到实现阶段才出问题
- **分节循环**：不会一次性丢出完整草稿——每节都是可审阅的小单元
- **恢复能力**：如果会话崩溃，智能体会在重启后重新读文件，并从第一个空白节继续
