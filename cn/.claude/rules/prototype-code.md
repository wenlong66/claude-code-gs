# 原型代码标准（放宽）

原型是用于验证想法的可丢弃代码。标准有意放宽以最大化迭代速度。目标是学习，而不是生产质量。

## 原型中允许的内容
- 硬编码值（不需要数据驱动配置）
- 最少或没有文档注释
- 简单架构（不需要依赖注入）
- 单例和全局状态
- 复制粘贴的代码（不需要抽象）
- 保留调试输出
- 占位符艺术和音频
- 快速而粗糙的解决方案

## 仍然需要的内容
- 每个原型位于自己的子目录中：`prototypes/[name]/`
- 每个原型必须有 `README.md`，包含：
  - 正在测试的假设
  - 如何运行原型
  - 当前状态（进行中 / 已结束）
  - 发现（原型结束时更新）
- 生产代码不得引用或导入 `prototypes/`
- 原型不得修改 `prototypes/` 外的文件
- 原型不得部署或发布

## 当原型成功时
如果原型验证了一个概念且功能进入生产阶段：
1. 原型代码不直接迁移——它按照生产标准重写
2. 原型 `README.md` 的发现为生产设计文档提供信息
3. 原型目录保留供参考但永不扩展

## 清理
在捕获发现后，应归档或删除已结束的原型。永远不要通过增量"清理"将原型代码发展为生产代码。
