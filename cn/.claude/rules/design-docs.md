# 设计文档规则

- 每个设计文档必须包含这 8 个部分：概述、玩家幻想、详细规则、公式、边缘情况、依赖、调优旋钮、验收标准
- 公式必须包含变量定义、预期值范围和示例计算
- 边缘情况必须明确说明会发生什么，而不是仅仅"优雅处理"
- 依赖必须是双向的——如果系统 A 依赖 B，B 的文档必须提及 A
- 调优旋钮必须指定安全范围以及它们影响什么游戏玩法方面
- 验收标准必须是可测试的——QA 测试员必须能够验证通过/失败
- 不要含糊其辞："系统应该感觉良好"不是有效的规范
- 平衡值必须链接到其源公式或理论基础
- 设计文档必须增量编写：先创建骨架，然后一次填写每个部分，各部分之间获得用户批准。将每个批准的部分立即写入文件以保留决策并管理上下文
