# Hook: post-sprint-retrospective

## 触发条件

在每个 sprint 结束时手动触发（通常由 producer 代理或人类开发者调用）。

## 目的

通过分析 sprint 数据自动生成回顾起点：计划与完成的对比、速度变化、bug 趋势和常见障碍。这不是 git hook，而是通过 `producer` 代理调用的工作流 hook。

## 实现

这是一个工作流 hook，不是 git hook。通过运行以下命令调用：

```
@producer Generate sprint retrospective for Sprint [N]
```

Producer 代理应该：

1. **读取 sprint 计划** 从 `production/sprints/sprint-[N].md`
2. **计算指标**：
   - 计划任务 vs 已完成任务
   - 计划故事点 vs 已完成故事点（如果使用）
   - 从上一个 sprint 结转的项目
   - 中途添加的新任务
   - 平均任务完成时间
3. **分析模式**：
   - 最常见的障碍
   - 哪个代理/领域有最多未完成工作
   - 哪些估计最不准确
4. **生成回顾**：

```markdown
# Sprint [N] 回顾

## 指标
| 指标 | 数值 |
|--------|-------|
| 计划任务 | [N] |
| 已完成任务 | [N] |
| 完成率 | [X%] |
| 从上一个 Sprint 结转 | [N] |
| 新添加任务 | [N] |
| 发现的 Bug | [N] |
| 修复的 Bug | [N] |

## 速度趋势
[Sprint N-2]: [X] | [Sprint N-1]: [Y] | [Sprint N]: [Z]
趋势: [改善 / 稳定 / 下降]

## 做得好的方面
- [自动检测：提前完成的任务]
- [主持人添加团队观察]

## 做得不好的方面
- [自动检测：结转或削减的任务]
- [自动检测：估计严重超标的领域]
- [主持人添加团队观察]

## 障碍
| 障碍 | 频率 | 解决时间 | 预防措施 |
|---------|-----------|----------------|-----------|

## 下一个 Sprint 的行动项
| # | 行动 | 负责人 | 优先级 |
|---|--------|-------|----------|

## 估计准确性
| 领域 | 平均计划 | 平均实际 | 准确性 |
|------|------------|-----------|----------|
```

5. **保存** 到 `production/sprints/sprint-[N]-retro.md`
