当此技能被调用时：

1. **解析参数**：
   - `version`：要为其生成说明的发布版本（例如 `1.2.0`）
   - `--style`：输出样式——`brief`（项目符号）、`detailed`（带上下文）、`full`（带开发者评论）。默认：`detailed`。

2. **从多个来源收集变更数据**：
   - 如果存在，读取 `production/releases/[version]/changelog.md` 中的内部变更日志
   - 运行上一个发布标签和当前标签/HEAD 之间的 `git log`
   - 阅读 `production/sprints/` 中的冲刺回顾以获取上下文
   - 阅读 `design/balance/` 中的任何平衡变更文档
   - 如果有，读取 QA 的缺陷修复记录

3. **将所有变更分类为面向玩家的类别**：
   - **新内容**：新功能、地图、角色、物品、模式
   - **游戏玩法变更**：平衡调整、机制变更、进度变更
   - **生活质量**：UI 改进、便利功能、可访问性
   - **缺陷修复**：按系统分组（战斗、UI、网络等）
   - **性能**：玩家可能注意到的优化改进
   - **已知问题**：对未解决问题的透明度

4. **将开发者语言翻译为玩家语言**：
   - "重构了伤害计算管线" → "改进了命中检测准确性"
   - "修复了库存管理器中的空引用" → "修复了打开库存时的崩溃"
   - "减少了战斗循环中的 GC 分配" → "改进了战斗性能"
   - 删除不影响玩家的纯内部变更
   - 保留平衡变更的特定数字（伤害：50 → 45）
