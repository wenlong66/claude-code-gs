name: onboard
description: "为加入项目的新贡献者或智能体生成上下文入职文档。总结项目状态、架构、约定和与指定角色或领域相关的当前优先级。"
argument-hint: "[role|area]"
user-invocable: true
allowed-tools: Read, Glob, Grep, Write
---

当此技能被调用时：

1. **阅读 CLAUDE.md** 了解项目概述和标准。

2. **如果指定了特定角色，从 `.claude/agents/` 阅读相关智能体定义**。

3. **扫描相关领域的代码库**：
   - 对于程序员：扫描 `src/` 了解架构、模式、关键文件
   - 对于设计师：扫描 `design/` 了解现有设计文档
   - 对于叙事：扫描 `design/narrative/` 了解世界构建和故事文档
   - 对于 QA：扫描 `tests/` 了解现有测试覆盖
   - 对于制作：扫描 `production/` 了解当前冲刺和里程碑

4. **阅读最近的变更**（如果有，使用 git log）以了解当前动力。

5. **生成入职文档**：

```markdown
# 入职：[角色/领域]

## 项目摘要
[2-3 句话总结这个游戏是什么及其当前状态]

## 你的角色
[这个角色在这个项目上做什么，关键职责，你向谁汇报]

## 项目架构
[与此角色相关的架构概述]

### 关键目录
| 目录 | 内容 | 你的交互 |
|-----------|----------|-----------------|

### 关键文件
| 文件 | 目的 | 阅读优先级 |
|------|---------|--------------|

## 当前标准和约定
[来自 CLAUDE.md 和智能体定义的相关约定摘要]

## 你领域的当前状态
[已经构建了什么，什么正在进行中，接下来计划什么]

## 当前冲刺上下文
[团队现在正在做什么，这个角色的期望是什么]

## 关键依赖
[这个角色最常交互的其他角色/系统]

## 常见陷阱
[这个领域的新贡献者容易犯的错误]

## 第一个任务
[建议的第一个任务以获得方向和提高效率]

1. [首先阅读这些文档]
2. [审查此代码/内容]
3. [从这个小型任务开始]

## 要问的问题
[新贡献者应该问的问题以完全定位]
```
