当此技能被调用时：

> **仅显式调用**：此技能只应在用户显式使用 `/launch-checklist` 请求时运行。不要基于上下文匹配自动调用。

1. **读取发布日期或 `dry-run` 模式的参数**。Dry-run 模式在不创建签字条目的情况下生成清单。

2. **收集项目上下文**：
   - 阅读 `CLAUDE.md` 了解技术栈、目标平台和团队结构
   - 阅读 `production/milestones/` 中的最新里程碑
   - 如果存在，阅读 `production/releases/` 中的现有发布清单
   - 如果存在，阅读 `design/live-ops/content-calendar.md` 中的内容日历

3. **扫描代码库健康状况**：
   - 统计 `TODO`、`FIXME`、`HACK` 注释及其位置
   - 检查生产代码中是否留有 `console.log`、`print()` 或调试输出
   - 检查占位符资源（搜索 `placeholder`、`temp_`、`WIP_`）
   - 检查硬编码的测试/开发值（localhost、测试凭据、调试标志）

4. **生成发布清单**：

```markdown
# 发布清单：[游戏标题]
目标发布：[日期或 DRY RUN]
生成日期：[日期]

---

## 1. 代码就绪

### 构建健康
- [ ] 所有目标平台上的干净构建
- [ ] 零编译器警告
- [ ] 所有单元测试通过
- [ ] 所有集成测试通过
- [ ] 性能基准测试在目标范围内
- [ ] 无内存泄漏（通过长时间浸泡测试验证）
- [ ] 构建大小在平台限制内
- [ ] 构建版本正确设置并标记在源代码控制中

### 代码质量
- [ ] TODO 计数：[N]（发布所需为零，或有记录例外）
- [ ] FIXME 计数：[N]（所需为零）
- [ ] HACK 计数：[N]（每个必须有记录在案的正当理由）
- [ ] 生产代码中无调试输出
- [ ] 无硬编码的开发/测试值
- [ ] 所有功能标志设置为生产值
- [ ] 错误处理覆盖所有关键路径
- [ ] 崩溃报告集成并验证

### 安全性
- [ ] 源代码中无暴露的 API 密钥或凭据
```
