当此技能被调用时：

## 1. 解析参数

三种模式：

- **完整规格**：`/setup-engine godot 4.6` — 提供了引擎和版本
- **仅引擎**：`/setup-engine unity` — 提供了引擎，将查找版本
- **无参数**：`/setup-engine` — 完全引导模式（引擎推荐 + 版本）

---

## 2. 引导模式（无参数）

如果未指定引擎，运行交互式引擎选择过程：

### 检查现有游戏概念
- 如果存在，读取 `design/gdd/game-concept.md` — 提取类型、范围、平台目标、美术风格、团队规模以及 `/brainstorm` 中的任何引擎推荐
- 如果不存在概念，通知用户：
  > "找不到游戏概念。考虑先运行 `/brainstorm` 来发现你想做什么 — 它也会推荐引擎。或者告诉我你的游戏，我可以帮你选择。"

### 如果用户想不借助概念进行选择，请问：
1. **什么类型的游戏？**（2D、3D 还是两者都有？）
2. **什么平台？**（PC、移动端、主机、Web？）
3. **团队规模和经验？**（个人初学者、个人有经验、小团队？）
4. **有强烈的语言偏好吗？**（GDScript、C#、C++、可视化脚本？）
5. **引擎许可预算？**（仅免费，还是商业许可也可以？）

### 提供推荐

使用此决策矩阵：

| 因素 | Godot 4 | Unity | Unreal Engine 5 |
|--------|---------|-------|-----------------|
| **最适合** | 2D 游戏、小型 3D、小型团队 | 移动端、中等规模 3D、跨平台 | AAA 3D、照片级真实感、大型团队 |
| **语言** | GDScript（+ C#、通过扩展的 C++） | C# | C++ / Blueprint |
| **成本** | 免费，MIT 许可 | 收入门槛下免费 | 收入门槛下免费，5% 特许权使用费 |
| **学习曲线** | 平缓 | 中等 | 陡峭 |
| **2D 支持** | 优秀（原生） | 良好（但以 3D 优先的引擎） | 可以但不理想 |
| **3D 质量上限** | 良好（快速改进中） | 非常好 | 同类最佳 |
| **Web 导出** | 是（原生） | 是（有限） | 否 |
| **主机导出** | 通过第三方 | 是（需要许可） | 是 |
| **开源** | 是 | 否 | 源代码可用 |

展示与用户回答相关的 1-2 个主要推荐及理由。让用户选择 — 永远不要强制推荐。
