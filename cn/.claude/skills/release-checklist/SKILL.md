当此技能被调用时：

> **仅显式调用**：此技能只应在用户显式使用 `/release-checklist` 请求时运行。不要基于上下文匹配自动调用。

1. **读取目标平台（`pc`、`console`、`mobile` 或 `all`）的参数**。如果未指定平台，默认为 `all`。

2. **阅读 CLAUDE.md** 了解项目上下文、版本信息和平台目标。

3. **从 `production/milestones/` 阅读当前里程碑** 以了解此发布应包含的功能和内容。

4. **扫描代码库** 查找待处理问题：
   - 统计 `TODO` 注释
   - 统计 `FIXME` 注释
   - 统计 `HACK` 注释
   - 记下它们的位置和严重性

5. **如果有的话，检查任何测试输出目录或 CI 日志中的测试结果**。

6. **生成发布清单**：

```markdown
## 发布清单：[版本] -- [平台]
生成日期：[日期]

### 代码库健康
- TODO 计数：[N]（如果很多则列出前 5 个）
- FIXME 计数：[N]（列出所有 -- 这些可能是阻塞因素）
- HACK 计数：[N]（列出所有 -- 这些需要审查）

### 构建验证
- [ ] 所有目标平台上的干净构建成功
- [ ] 无编译器警告（零警告策略）
- [ ] 所有资源已包含并正确加载
- [ ] 构建大小在预算内（[目标大小]）
- [ ] 构建版本号正确设置（[版本]）
- [ ] 构建可从标记的提交重现

### 质量门
- [ ] 零 S1（严重）缺陷
- [ ] 零 S2（主要）缺陷 -- 或有 producer 批准的记录例外
- [ ] 所有关键路径功能已测试并由 QA 签字
- [ ] 性能在预算内：
  - [ ] 最低规格硬件上达到目标 FPS
  - [ ] 内存使用在预算内
  - [ ] 加载时间在预算内
  - [ ] 长时间游戏会话中无内存泄漏
- [ ] 与上一个构建无回归
- [ ] 浸泡测试通过（4+ 小时连续游戏）
```
